# Content Guide

## 一套攻略的推荐结构

1. 我的地图：先告诉观众这趟攻略覆盖哪些区域。
2. 街区地图：解释为什么不是按景点清单走。
3. 核心路线：经典地标、必去博物馆、主要动线。
4. 街区体验：住处、咖啡、店面、路上的细节。
5. 吃和日常：面包店、超市、自煮、普通正餐。
6. 个人标签：运动、偶遇、临时决定、小众体验。
7. 交通和体力提醒：用少量画面解释节奏安排。
8. 收束：用一句话总结这趟旅行的方法论。

## 新城市开稿流程

建议按这个顺序处理，不要一上来就排照片墙：

1. 先放 1-3 张地图：总览地图、街区地图、交通/跨区地图。
2. 按地图把照片分组：核心景点、街区乱逛、吃和日常、个人标签、交通提醒。
3. 每组先选 5-8 张最有讲头的图，宁可多页，也不要把图挤成小墙。
4. 每页只写短标题和口播提示，文字服务于录制，不抢照片。
5. 先用默认 SVG 贴纸排版，等顺序稳定后再换透明 PNG 定制贴纸。

伊斯坦布尔可以这样分：

1. 总览地图：欧洲侧、亚洲侧、海峡、住处位置。
2. 老城核心：圣索菲亚、蓝色清真寺、地下水宫、大巴扎周边。
3. 海峡和渡轮：不要只讲景点，讲城市被水切开的体感。
4. 贝伊奥卢/加拉塔：坡、街、咖啡、店、夜色。
5. 亚洲侧：卡德柯伊、街区生活、吃饭和闲逛。
6. 猫、茶、甜点、市场：做成生活感段落，不要变成景点清单。
7. 交通和体力：坡路、轮渡、拥挤、跨区时间。
8. 收束：这座城不是单线刷完，而是在两片大陆之间慢慢晃。

## Slide 字段

```js
{
  kind: "map",
  section: "01 / 我的地图",
  src: "maps/core-map.jpg",
  title: "先从这张图开始",
  text: "给自己看的口播提示。",
  note: "更细的提醒，也默认不显示。",
  tag: "地图"
}
```

字段说明：

- `kind`: 可选。`map` 会隐藏路线装饰层，让地图更干净。
- `section`: 页签。建议格式：`03 / 第一站`。
- `src`: 相对于 `assetsBase` 的路径。
- `title`: 画面上显示的短标题。
- `text`: 口播提示，默认不显示。
- `note`: 更细提醒，默认不显示。
- `tag`: 主题标签，默认不显示。
- `position`: 可选。控制图片位置，例如 `"50% 40%"`。
- `sticker`: 可选。竖屏版的小人贴纸配置，只做视觉辅助，不要写大字盖住原图。

贴纸建议：

- 地图页不放贴纸，先让路线说清楚。
- 照片页可以放唐小蓝贴纸，道具跟照片主体相关，例如铁塔、咖啡、面包、地铁。
- 有透明 PNG 贴纸时，在 `sticker.src` 填路径；没有时模板会用 `prop`、`mood`、`pose` 生成轻量 SVG 贴纸。
- 位置尽量错开照片主体和人脸，保持“照片是主角，贴纸是旁白”。
- 不在贴纸里写长句，也不把贴纸当评论区。

贴纸复用策略：

- 通用动作复用：拿地图、指路、避坑、疲惫、惊喜、喝咖啡、赶车。
- 城市道具定制：伊斯坦布尔可以补清真寺穹顶、渡轮、茶杯、猫、坡路、香料市场、鱼三明治、土耳其甜点。
- 同一城市的定制贴纸控制在 6-12 张，避免每页都重新生成导致风格散。
- 横图优先把贴纸放在图片下方空白区；竖图优先放低信息角落，不挡人脸、文字、主体建筑。
- `sticker.src` 留空时用模板 SVG 快速占位；最终录制前再换成 `assets/stickers/custom/final/` 里的透明 PNG。

## 网页资料怎么放

如果攻略里有网页：

1. 截图保存到 `assets/web/`。
2. 在 `sources` 增加链接。
3. 在 `slides` 里像照片一样引用截图。

```js
sources: [
  {
    title: "博物馆官网",
    url: "https://example.com",
    note: "门票和开放时间来源"
  }
],
slides: [
  {
    section: "08 / 网页资料",
    src: "web/museum-hours.jpg",
    title: "开放时间先看官网"
  }
]
```

## 录制建议

- 竖屏：1080 x 1920。
- `index.html` 更像地图 App 讲解页。
- `vertical.html` 更适合录短视频：只保留内容画布，不带平台 UI，也不在原图上盖评论。
- 浏览器隐藏书签栏更干净。
- 口播跟着照片讲，不要念页面文字。
- 每个地点先讲地图，再展开照片。
