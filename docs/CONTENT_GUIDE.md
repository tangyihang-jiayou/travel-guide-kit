# Content Guide

## 一套攻略的推荐结构

1. 我的地图：先告诉观众这趟攻略覆盖哪些区域。
2. 街区地图：解释为什么不是按景点清单走。
3. 核心路线：经典地标、必去博物馆、主要动线。
4. 街区体验：住处、咖啡、店面、路上的细节。
5. 吃和日常：面包店、超市、自煮、普通正餐。
6. 个人标签：运动、偶遇、临时决定、小众体验。
7. 交通和体力提醒：用少量画面解释节奏安排。
8. 收束：用一句话总结这趟旅行的方法论。

## Slide 字段

```js
{
  kind: "map",
  section: "01 / 我的地图",
  src: "maps/core-map.jpg",
  title: "先从这张图开始",
  text: "给自己看的口播提示。",
  note: "更细的提醒，也默认不显示。",
  tag: "地图"
}
```

字段说明：

- `kind`: 可选。`map` 会隐藏路线装饰层，让地图更干净。
- `section`: 页签。建议格式：`03 / 第一站`。
- `src`: 相对于 `assetsBase` 的路径。
- `title`: 画面上显示的短标题。
- `text`: 口播提示，默认不显示。
- `note`: 更细提醒，默认不显示。
- `tag`: 主题标签，默认不显示。
- `position`: 可选。控制图片位置，例如 `"50% 40%"`。

## 网页资料怎么放

如果攻略里有网页：

1. 截图保存到 `assets/web/`。
2. 在 `sources` 增加链接。
3. 在 `slides` 里像照片一样引用截图。

```js
sources: [
  {
    title: "博物馆官网",
    url: "https://example.com",
    note: "门票和开放时间来源"
  }
],
slides: [
  {
    section: "08 / 网页资料",
    src: "web/museum-hours.jpg",
    title: "开放时间先看官网"
  }
]
```

## 录制建议

- 竖屏：1080 x 1920。
- `index.html` 更像地图 App 讲解页。
- `vertical.html` 更适合录短视频：只保留内容画布，不带平台 UI，也不在原图上盖评论。
- 浏览器隐藏书签栏更干净。
- 口播跟着照片讲，不要念页面文字。
- 每个地点先讲地图，再展开照片。
